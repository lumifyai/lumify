[Lumify](https://lumify.ai) is an agent-ready sports intelligence API: schedules,
live scores, odds, public betting splits, and explainable AI bet confidence
(with rationale) across 8+ sports. The `langchain-lumify` package loads Lumify's
full tool surface into any LangChain or LangGraph agent via its hosted
[MCP](https://modelcontextprotocol.io) server.

## Overview

### Integration details

| Class | Package | Serializable | JS support | Package latest |
| :--- | :--- | :---: | :---: | :---: |
| `get_lumify_tools` / `LumifyToolkit` | `langchain-lumify` | ❌ | ❌ (no dedicated JS package; see below) | ![PyPI - Version](https://img.shields.io/pypi/v/langchain-lumify) |

`langchain-lumify` itself is Python-only. JavaScript/TypeScript users get the
same tools directly from `@langchain/mcp-adapters` pointed at
`https://lumify.ai/mcp`. See the [agent cookbook](https://lumify.ai/docs/agent-cookbook.md#5-langchain--langgraph-mcp-adapters)
for a copy-paste example.

### Tool features

Loads 17 tools: `list_sports`, `list_seasons`, `list_events`, `get_event`,
`batch_get_events`, `query_events`, `get_live_score`, `get_odds`,
`get_odds_history`, `get_splits`, `get_intelligence`, `list_teams`, `get_team`,
`search_players`, `get_player`, `get_player_events`, `estimate_cost`. All are
read-only and return structured JSON.

## Setup

Install the package:

```bash
pip install langchain-lumify
```

### Credentials

Get a **free instant key in seconds (no signup, email, or card required)** at
[lumify.ai/docs/ai](https://lumify.ai/docs/ai) (100 credits, 14-day expiry), or
create a persistent account key at [lumify.ai/api-keys](https://lumify.ai/api-keys).

```python
import getpass
import os

if not os.environ.get("LUMIFY_API_KEY"):
    os.environ["LUMIFY_API_KEY"] = getpass.getpass("Lumify API key:\n")
```

## Instantiation

```python
from langchain_lumify import get_lumify_tools

tools = await get_lumify_tools()  # requires LUMIFY_API_KEY (or api_key=...)
```

`get_lumify_tools` requires a resolvable API key by default so the first tool
call does not fail with an opaque 401. Pass `require_api_key=False` only to
introspect the public `tools/list` catalog. Raw `lmfy-...` values and
`Bearer lmfy-...` copy-pastes are both accepted.

## Invocation

```python
get_intelligence = next(t for t in tools if t.name == "get_intelligence")
result = await get_intelligence.ainvoke({"event_id": 12345})
```

## Use within an agent

```python
import asyncio
from langchain.agents import create_agent
from langchain_lumify import get_lumify_tools


async def main():
    tools = await get_lumify_tools()
    agent = create_agent("openai:gpt-4.1", tools)
    return await agent.ainvoke(
        {"messages": "What's the best MLB bet today, and why?"}
    )


asyncio.run(main())
```

## API reference

- `get_lumify_tools(api_key=None, url="https://lumify.ai/mcp", *, require_api_key=True)`
  returns the list of Lumify tools as LangChain `BaseTool`s.
- `LumifyToolkit.acreate(api_key=None, url=..., *, require_api_key=True)` is an
  async constructor returning a toolkit whose `get_tools()` yields the same
  tools. Direct `LumifyToolkit()` with no tools raises.
- `MissingApiKeyError` is raised when a key is required but none resolves.

Source: [github.com/lumifyai/lumify](https://github.com/lumifyai/lumify).
MCP endpoint: `https://lumify.ai/mcp`
